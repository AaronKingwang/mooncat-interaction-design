# B端典型场景速查（落地手册）

> 参考四本经典的思想方向，落到 B端最常见的页面/场景上。每个场景给「目标 → 关键要点 → 常见坑 → 参考方向」。设计或评审对应页面时，直接查本册。

## 1. 列表页（B端最核心页面）

**用户目标**：快速找到目标数据 / 批量处理 / 监控状态。

✅ 要点
- **默认服务主任务**：默认排序（如待处理优先/时间倒序）、默认筛选（如「我的/进行中」）、默认列为高频列。
- **筛选区分层**：常用筛选直接露出；高级筛选折叠（隐藏策略）；支持「保存的筛选方案」（转移给专家）。
- **列精简+可配**：首屏不横向滚动；非核心列进「列设置」。数字右对齐、状态用 Tag、时间统一格式。
- **行内高频操作**：查看/编辑/审批等放行内；低频进「···更多」。可就地编辑的别跳详情（去 excise）。
- **批量操作**：选中出现批量条，显示「已选 N 项」，支持跨页全选与清除；危险批量给影响范围确认。
- **状态记忆**：记住筛选、排序、列宽、分页、上次停留——用户回来时接着干。
- **分页**：显示总数、可跳页；核对型工作优于无限滚动。

⚠️ 常见坑
- 把后台所有字段全塞进列表 → 用「删除+隐藏」精简。
- 每次进来都要重设筛选 → 加默认值与记忆。
- 操作藏在详情页里，列表看不了也改不了 → 内联化。
- 操作后跳回第一页、丢筛选 → 保持上下文。

📖 Cooper(去excise/保持流)、Colborne(删除/隐藏/转移)、Tidwell(数据表格)、Krug(扫读/视觉层级)

## 2. 表单页 / 编辑页

**用户目标**：准确、快速地录入或修改数据。

✅ 要点
- **分区+小标题**：按业务心智分组，别一长条到底（组织策略）。
- **减字段**：能自动带出的别让用户填；能合并的合并（删除策略 + 转移给系统）。
- **智能默认值**：默认当前时间/本人/常用项，90% 场景不用改。
- **控件选对**：见 designing-interfaces §3.2（少选项用单选、多选项用带搜索下拉…）。
- **即时校验+防错**：就近即时提示，别等提交后一次报一堆；非法项直接禁用/约束格式。
- **必填清晰**、单位/示例就近（别只靠 placeholder，它一输入就消失）。
- **保存反馈明确**：保存中 loading、成功 toast、失败说清哪错怎么改。
- **防丢失**：长表单未保存离开给提醒；支持草稿/自动保存更佳。

⚠️ 常见坑
- 40 个字段一屏平铺、无分组 → 分区。
- 提交后弹一堆红字 → 改即时校验。
- 关键说明放 placeholder，输入后没了 → 改常驻说明。
- 用技术字段名当标签 → 换业务语言。

📖 Cooper(数据录入/防错)、Colborne(删除/组织/转移)、Tidwell(表单/控件)、Krug(文案/别让我想)

## 3. 详情页

**用户目标**：了解一个对象的全貌 / 在此对象上做操作。

✅ 要点
- **信息按重要度分层**：关键状态与标识置顶；次要信息折叠或分 Tab。
- **面包屑 + 明确返回**：回答「我在哪、怎么回去」。
- **操作就近**：针对该对象的操作放显眼处，主次分明（一个主按钮）。
- **关联信息内联展示**：相关子数据在本页看到，别再层层跳转（去 excise）。
- **审阅型场景用「左列表+右详情」**：切换不跳页，适合逐条审。

⚠️ 常见坑：信息平铺无层级；操作散落；为看关联数据反复跳页。

📖 Cooper(去excise/心理模型)、Tidwell(概览+明细)、Krug(我在哪)

## 4. 仪表盘 / 工作台

**用户目标**：概览全局、发现异常、进入待办。

✅ 要点
- **最重要指标在左上**（视觉起点）；一屏看全核心，别让用户滚动找关键数。
- **为决策/行动服务**：每个指标能下钻到明细；异常可直达处理入口。
- **按角色定制**：老板看趋势、主管看团队、操作员看待办——别一个看板给所有人。
- **图表按问题选**：趋势→折线、占比→环/饼、对比→柱、构成→堆叠；别为炫技上复杂图。
- **待办/提醒模块**：把「今天该做什么」直接推到用户面前（转移给系统）。

⚠️ 常见坑：堆一堆炫酷图表但不能下钻、不指向行动；所有角色同一看板；关键数字被装饰淹没。

📖 Tidwell(仪表盘/图表)、Cooper(目标导向)、Krug(视觉层级)

---

## 5. 筛选 / 搜索

✅ 要点
- **常用直出、高级折叠**（隐藏）；筛选维度按业务问题设计，不是把字段堆上去。
- **保存常用筛选方案**（转移给专家）；显示当前生效的筛选条件（可单独清除某项）。
- **即时反馈结果数**；无结果时给「放宽条件」建议而非空白。
- 搜索支持模糊/多关键词，宽容输入（接受多种格式，系统去规整）。

⚠️ 常见坑：十几个筛选条件平铺；筛完不知道当前筛了啥；无结果只给空白页。

📖 Colborne(隐藏/转移)、Cooper(数据检索/宽容)、Tidwell(筛选模式)

---

## 6. 批量操作

✅ 要点
- 选中后出现**批量动作条**，常显「已选 N 项」。
- 支持**全选/跨页全选/清除选择**，并明确区分「本页全选」与「全部 N 条」。
- 批量结果给**逐条反馈**（成功 X 条、失败 Y 条及原因），别只说「操作成功」。
- 危险批量：确认框写清**影响范围与不可逆性**（见 about-face 撤销优于确认）。

⚠️ 常见坑：不显示已选数量；跨页全选语义不清；批量失败不说哪几条为什么。

📖 Tidwell(批量动作)、Cooper(撤销/反馈)

---

## 7. 权限 / 角色差异

✅ 要点
- **界面随角色变**：无权限的功能**隐藏或置灰+说明**，别让用户点了才报「无权限」。
- 置灰要给原因（hover 提示「需管理员权限」），别让用户对着灰按钮猜。
- 不同角色的默认视图/密度/入口不同（操作员要效率，管理员要配置，老板要概览）。
- 权限配置界面本身用**树/矩阵**表达层级与继承关系。

⚠️ 常见坑：所有角色一套界面；点了才报无权限；权限配置页看不懂继承关系。

📖 Cooper(角色/目标)、Colborne(为主流用户)、Tidwell(树表)

## 8. 五种边界状态（最容易被漏掉）

B端评审必查的五态，缺一即扣分：

| 状态 | 要点 |
|------|------|
| **空状态** | 说清「为什么空 + 下一步做什么」（如「暂无数据，点击新建」），别只留空白 |
| **加载中** | >0.5s 给 loading；长任务给进度与预期；用骨架屏减少跳动 |
| **错误/异常** | 人话说清哪错、为什么、怎么办；给重试/返回；不暴露堆栈码 |
| **无权限** | 隐藏或置灰+原因；给「如何申请」路径 |
| **极端数据** | 超长文本截断+悬浮全文；超大数量分页/虚拟滚动；0/负数/超大数字格式化 |

📖 Tidwell(空状态/进度)、Cooper(容错)、Krug(将就着用也不出错)

## 9. 反馈与确认（贯穿所有场景）

✅ 要点
- **每个操作都有反馈**：loading → 结果（成功/失败）。无声成功也会让用户怀疑「点到没」。
- **可逆操作用「执行+撤销」**，不要用确认框骚扰（撤销优于确认）。
- **不可逆/高危才确认**，且写清影响范围；极高危加「输入确认」摩擦。
- 反馈**就近出现**：字段错误在字段旁，全局结果用 toast，别让用户找。

📖 Cooper(反馈/撤销/流)、Tidwell(toast/inline error)

## 10. 一页速记：从原理到场景的映射

| 你在设计… | 先想的一句话 |
|-----------|-------------|
| 列表页 | 主任务是什么？默认就摆成它最省事的样子 |
| 表单页 | 哪些字段能删、能自动填？剩下的分组、即时校验 |
| 详情页 | 关键信息置顶，关联数据内联，别让人跳来跳去 |
| 仪表盘 | 谁看？看完要做什么决定/行动？指向它 |
| 筛选 | 常用直出、高级折叠、能保存 |
| 批量 | 已选几条、影响多大、失败是哪几条 |
| 权限 | 界面随角色变，没权限的别让人点了才知道 |
| 任何页 | 空/载/错/无权/极端——五态都设计了吗？ |
