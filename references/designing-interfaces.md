# B端界面模式库（B端适配）

> 本册**参考**《Web界面设计 / Designing Interfaces》（Jenifer Tidwell）的思想方向——**不要发明轮子，复用被验证过的界面模式**——整理出 B端最常用的模式与选用要点。
> **来源声明**：以下为参考该书思路并结合 B端实践的整理，**非原书原文提炼**；文中书名/作者仅表「参考方向」。

---

## 0. 用模式的原则

- 模式是「反复出现的问题 + 被验证的解法」。用对模式 = 用户不必学习。
- 选模式先看**内容结构**和**用户任务**，而非好不好看。
- 同一问题有多个模式时，按「用户熟悉度 + 场景匹配度」选，不选最炫的。

---

## 1. 组织内容：导航与信息架构

| 模式 | 用于 | B端要点 |
|------|------|--------|
| **左侧导航（Vertical Nav）** | 模块多、层级深的后台 | B端主流。一级图标+文字，二级展开；当前项高亮；支持收起省空间 |
| **顶部标签（Tabs）** | 同一对象的多个视图/维度 | Tab 数控制在 7±2；不放并列的独立功能（那是导航不是 Tab） |
| **面包屑（Breadcrumbs）** | 层级深的详情/配置页 | 回答「我在哪」；每级可点回跳 |
| **向导（Wizard）** | 低频、线性、步骤强依赖的复杂任务（初始化配置、开户） | 显示步骤条与当前步；可返回改；高频任务别用向导（太慢） |
| **仪表盘（Dashboard）** | 概览、监控、决策入口 | 最重要指标在左上；一屏看全局；可下钻到明细 |

**导航设计判据**：用户在任何页面都能答出「这是什么系统、我在哪、能去哪、怎么回去」。

## 2. 展示复杂数据：B端的重头戏

### 2.1 数据表格（Data Table / Grid）——B端第一控件

- **列**：只放当前任务需要的列（用「删除」策略），其余进「列配置」（隐藏策略）。首屏别横向滚动。
- **排序**：可点表头排序；默认排序服务主任务（如按时间倒序、按待处理优先）。
- **行操作**：高频操作放行内按钮/图标；低频收进行尾「···更多」。
- **对齐**：文本左对齐、数字右对齐、状态用标签(Tag)、时间统一格式。
- **大数据量**：分页 vs 无限滚动——B端偏「分页+可跳页+显示总数」，便于定位与核对。
- **行内编辑（Inline Edit）**：高频修改用就地编辑，省去进详情页（去 excise）。
- **固定表头/首列**：长表格滚动时保持列头与关键标识列可见。

### 2.2 其他复杂数据模式

| 模式 | 用于 |
|------|------|
| **树表 / 树形（Tree Table）** | 有层级的数据（组织架构、分类、菜单权限） |
| **卡片列表（Card / Grid of Thumbnails）** | 视觉对象为主（图片、商品、模板） |
| **看板（Kanban）** | 状态流转的任务（工单、审批、需求池） |
| **概览+明细（Overview + Detail）** | 左列表右详情，切换不跳页，适合审阅型工作 |
| **主从（Master-Detail）/ 列表-表单同屏** | 边看列表边编辑 |
| **数据可视化（图表）** | 趋势看折线、占比看饼/环、对比看柱、分布看散点——按问题选图，别为炫技上图 |

## 3. 获取用户输入：表单与控件

### 3.1 表单结构（Form）
- **分组（Titled Sections）**：按业务含义把字段分块加小标题，别一长条到底。
- **标签对齐**：B端长表单推荐「标签左对齐、右侧输入」利于扫读与紧凑；或顶对齐利于快速录入。
- **必填/可选**：明确标注；能减字段就减（简约·删除）。
- **就近帮助**：说明文字、示例、单位就近放，别用会消失的 placeholder 承载关键信息。

### 3.2 控件选择（控件决策）
| 场景 | 用什么 | 别用什么 |
|------|-------|---------|
| 2 个互斥选项 | 开关/单选 | 下拉（多一次点击） |
| 3-7 个互斥选项 | 单选按钮组（一眼看全） | 下拉 |
| >7 个互斥选项 | 下拉/带搜索的选择器 | 一堆单选 |
| 多选少量 | 复选框组 | 多选下拉 |
| 多选大量 | 穿梭框(Transfer)/带搜索多选 | 一屏复选框 |
| 日期/范围 | 日期选择器/区间选择器 | 手输文本 |
| 数值范围 | 步进器/滑块 | 纯文本 |

### 3.3 输入辅助模式
- **自动补全（Autocomplete）**、**输入提示（Input Hints）**、**结构化格式（Input Prompt）**、**好的默认值（Good Defaults）**、**即时校验（Forgiving Format + inline validation）**。
- **防错优于报错**：不合法的选项禁用/不可选，而非让用户选了再报错。

## 4. 命令与动作（Actions）

- **主次分明**：一个区域一个主按钮（Primary，实心/主色），其余次要（描边/文字按钮）。别满屏都是主色按钮。
- **按钮文案 = 动词短语**：「保存并发布」「导出Excel」，而非「确定」「提交」。让用户一眼看出「点了会发生什么」。
- **批量操作**：选中行后出现批量动作条；显示「已选 N 项」；提供全选/跨页全选与清除。
- **危险操作**：视觉降权（放次要位/红色文字），并配确认或撤销（见 about-face 撤销优于确认）。

---

## 5. 反馈与状态

| 模式 | 用于 |
|------|------|
| **即时反馈（Progress / Spinner）** | 任何 >0.5s 的操作，给 loading；>几秒给进度与预期 |
| **轻提示（Toast / Snackbar）** | 操作成功/失败的非阻断反馈，可带「撤销」 |
| **内联错误（Inline Error）** | 表单字段就近报错，指明哪错、怎么改 |
| **空状态（Empty State）** | 无数据时给「原因+下一步动作」，别只留空白 |
| **骨架屏（Skeleton）** | 加载中保持布局稳定，减少跳动 |

---

## 6. 让它好看（Visual Style，服务于可用性）
- **视觉层级**：用大小/粗细/颜色/留白建立主次，让重要的先被看到。
- **对齐与网格**：统一栅格，减少视觉噪音（B端信息密集，对齐是秩序感来源）。
- **克制用色**：主色只用于关键动作与强调；状态色（成功绿/警告橙/危险红）语义固定不乱用。
- **一致性**：同类元素同样式、同位置、同交互——一致性本身就降低学习成本。

---

## 7. 选型速查
- 展示一堆结构化数据 → **数据表格**（+列配置+排序+行内编辑）
- 状态流转的任务 → **看板** 或 **表格+状态筛选**
- 审阅型工作（逐条看+改） → **概览+明细（左右分栏）**
- 有层级的数据 → **树 / 树表**
- 低频线性复杂配置 → **向导**
- 高频复杂配置 → **单页分区表单 + 智能默认**，不要向导
- 概览监控 → **仪表盘**（要点见 bend-scenarios）
